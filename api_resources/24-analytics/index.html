Analytics

/environments/{environmentName}/stats
Provides metrics gathered on APIs and related entities in Apigee
Apigee collects statistic on API traffic as it flow from apps to APIs. Statistics collected include API call information (like URL, IP, and User id) latency, errors, and so on. You can add other information by defining policies that identify message elements of interest for collection. For example, statistics can be collected from message headers, query parameters, portions of the request and response extracted from XML or JSON, and so on. The statistics collected are used to generate out-of-the-box reports. You can also generate reports tailored to your needs by tracking key value pairs specific to your traffic. Apigee Analytics helps you to view long-term usage trends. You can segment your audience by top developers and apps. You can understand the usage by using the API method to know where to invest, and create custom reports on business-level information. All data is pushed to Analytics where it is persisted by the Analytics server in the background. Data aggregation jobs compile various built-in or custom reports: Usage by API (built-in) Usage by app (built-in) Usage developer (built-in) Usage by API Traffic (built-in) Usage by API Product (built-in) Usage by Environment (built-in) Usage by Customizing the API
