Environments

/environments
A deployment environment for APIs
By default, Apigee Enterprise Cloud organizations have two environments: 'test' and 'prod'. An environment provides a runtime execution context for APIs. An API Proxy revision must be deployed to an environment before it can be accessed at runtime. No constraints are placed on the usage of test versus prod, for example. Developers are free to implement and enforce and type or testing, promotion, and deployment procedures that suit their development lifecycle.
