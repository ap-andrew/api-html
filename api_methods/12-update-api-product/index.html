Update API Product :: /apiproducts/{apiproductName}

API resource: API Products

Verb: PUT

Doc: Call PUT on an existing API Product to update the API Product. You are only required to include the elements to be updated in the update request.

Description: Updates an existing APi Product

cURL: 
curl -u {myname:mypass} "https://api.enterprise.apigee.com/v1/o/{myorg}/apiproducts"
Request Payload: 
       {
  "approvalType" : "auto",
  "attributes" : [ {
    "name" : "RateLimit",
    "value" : "200"
  }, {
    "name" : "Threshold",
    "value" : "25"
  } ],
  "displayName" : "Sandbox Silver1",
   "apiResources" : [ "/v1/**", "/v3/receiveSMS", "/v2/*/send" ]
}
Request Payload Elements:
Name Description Default Required?
approvalType Manual or Auto. If Manual, consumer key is generated and returned in 'pending' state. If manual consumer key generated and returned in 'Approved' state. N/A No Detail
attributes Arbitrary name/value pairs, most commonly used to store variables that define Rate Limit (Quota or Spike Arrest) settings for an API Product (otherwise known as a Service Plan) N/A No Detail
displayName The display name of the API Product, used in Apps that display the API Product to Developers N/A No Detail
apiResources A comma separated list of API resources to be bundled in the API Product. N/A No Detail

  {

  "approvalType" : "auto",
  "attributes" : [ {
    "name" : "RateLimit",
    "value" : "200"
  }, {
    "name" : "Threshold",
    "value" : "25"
  } ],
  "createdAt" : 1343052029205,
  "createdBy" : "Admin",
  "displayName" : "Sandbox Silver1",
  "lastModifiedAt" : 1343052029692,
  "lastModifiedBy" : "Admin",
  "name" : "sandbox",
  "scopes" : [ ]
}
Response Payload Elements:
Name Description
apiResources: A list of URIs (including wildcards) that represent the bundle of APIs permitted to consumers of this API Product Detail
approvalType: Determines the state of the consumer key generated for requesters of this API Product. For Auto, key are generated in "Approved' state. For Manual, keys are approved in 'Pending' state. Detail
attributes: Any app-specific arbitrary attributes that may be used to define visibility of the API Product or runtime policy enforcement (such as setting for a rate-limiting policy. Detail
createdAt: Unix time Detail
createdBy: Apigee user account name of the individual who submitted the POST request that created this entity Detail
lastModifiedAt: Unix time Detail
lastModifiedBy: Apigee user account name of the individual who last modified this entity Detail
name: Name of the API Product Detail
scopes: Oauth scopes required in the Access Token(s) used to consime any of the API Reqsources (URIs) bundled in this API Product Detail
Errors 
  {
  "code" : "keymanagement.service.apiproduct_doesnot_exist",
  "message" : "ApiProduct with name sandbox1 does not exist",
  "contexts" : [ ]
}
Auth: Basic Auth
