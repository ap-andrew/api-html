Deploy an API to an Environment :: /apis/{apiName}/revisions/{revisionNumber}/deployments?action=deploy&env={envName}&basepath={/basepath}

API resource: APIs

Verb: POST

Doc:

Description: Pushes an API to an environment (typically 'test' or 'prod') for execution

cURL: 
curl -u myname:mypass https://api.enterprise.apigee.com/v1/o/{myorg}/apis/{apiName}/revisions/{revisionNumber}/deployments?action=deploy&env={envName}&basepath={/basepath}

Response Payload Elements:
Name Description
aPIProxy: The name of the API proxy Detail
configuration: The basepath for this deployment (appended to the basepath defined in the Proxy Endpoint if any) Detail
steps: Any policy steps attached to the API proxy Detail
environment: The environment where the API proxy is deployed ('test' or 'prod') Detail
name: The revision name (always a number) Detail
organization: The API provider's Apigee organization name Detail
revision: The number of the revision Detail
server: A set of servers deployed in support of the API proxy (primarily for Apigee support) Detail
state: The state of the API proxy, either 'deployed' or 'undeployed' Detail
Errors
Auth: Basic Auth
